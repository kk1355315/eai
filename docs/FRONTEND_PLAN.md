# AI 冰箱前端项目计划书

## 一、项目目标

本项目要做一个移动端优先的“冰箱健康系统”前端，用于展示树莓派自动识别后的水果库存、保存状态、今日推荐和基础用户信息。

第一版目标不是做复杂 App，而是完成一个能展示、能联调、能用于 Demo 的高质感前端。

```text
树莓派识别水果
  ↓
后端写入库存
  ↓
前端展示库存、剩余天数、今日推荐、提醒状态
```

## 二、产品定位

产品名称：AI 冰箱 / 冰箱健康系统

使用场景：

- 用户打开前端查看当前冰箱里有什么水果
- 查看哪些水果还新鲜
- 查看哪些水果需要优先吃
- 查看今日推荐水果
- 后续接入 AI 饮食建议和用户健康偏好

目标用户：

- 家庭用户
- Demo / 答辩 / 项目展示观众
- 后续可扩展到智能冰箱、健康饮食管理场景

## 四、设计方向

视觉风格：Apple / iOS 26 Liquid Glass 风格的 Web 模拟版。

需要注意：Web 前端不能直接调用 iOS 26 原生 Liquid Glass / SwiftUI `glassEffect()`。真正的系统级材质只有原生 Apple 平台组件能获得。Web 端只能模拟以下特征：

- 控制层浮在内容之上
- 半透明玻璃材质
- 背景模糊和亮度调整
- 细边缘高光
- 内阴影与外阴影
- 指针 / 触摸动态高光
- 底部导航有明显玻璃浮层感
- 内容层保持克制，避免全页面花哨透明

设计原则：

- 背景克制，不使用彩带和装饰性渐变
- 内容卡片尽量清晰，保证中文可读性
- Liquid Glass 重点用于底部导航、按钮、状态胶囊和浮层
- 页面整体像 iOS 系统 App，而不是营销网页

### 动态效果范围

第一版只加入克制、必要的动态效果，用来增强 Liquid Glass 的“活性”，不做过多动画堆叠。

需要实现：

- 底部导航切换时，选中玻璃块在“首页 / 库存 / 我的”之间平滑滑动
- 鼠标悬停在主要窗口或卡片上时，窗口轻微上浮、放大，并加强玻璃高光和阴影

暂不实现：

- 页面切换淡入上浮
- 库存进度条加载动画
- 按钮点击压缩和回弹
- 大幅度液态形变
- 复杂滚动视差

动效原则：

- 动效要短，控制在 200ms 到 300ms 左右
- 放大幅度要小，避免页面跳动或遮挡文字
- 鼠标悬停效果只针对桌面端生效，触屏端不强制触发
- 支持 `prefers-reduced-motion`，用户关闭动效时应减少或关闭动画
- 动态效果服务于质感，不喧宾夺主

## 六、数据来源

前端优先调用后端接口：

```text
GET /inventory
GET /advice/today
GET /profile
```

当前第一版主要使用：

```text
GET /inventory
```

库存字段映射：

```text
food.model_label         水果类型
food.display_name        后端显示名
confirmed_quantity       确认数量
unit                     单位
days_stored              已存放天数
remaining_days           剩余天数
storage_state            保存状态
status                   库存状态
```

前端展示规则：

```text
storage_state = fresh
显示：新鲜

storage_state = eat_soon
显示：还剩 n 天 / 优先吃

storage_state = check_required
显示：待确认

storage_state = not_recommended
显示：不推荐
```

首页推荐排序：

```text
1. eat_soon 优先
2. remaining_days 更少的优先
3. fresh 其次
4. check_required 不作为直接推荐食用，只提示确认
```

## 十、风险与注意事项

1. Web 无法完全复刻 iOS 26 原生 Liquid Glass。
   原生效果依赖 Apple 系统渲染、SwiftUI / UIKit 材质和真实设备能力。Web 只能模拟视觉和交互。

2. Liquid Glass 不能滥用。
   Apple 官方也强调它更适合控制层和导航层。如果所有内容卡片都强玻璃化，会降低可读性。

3. 后端数据可能为空。
   前端需要保留兜底数据、空状态和错误状态。

4. 树莓派识别结果需要稳定写入后端。
   前端不负责识别，只依赖后端库存结果。如果树莓派没有写入，前端就没有真实库存。
