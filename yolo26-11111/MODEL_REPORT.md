# YOLO11n 水果检测模型汇报材料

## 1. 模型与数据集

- 模型：`YOLO11n` 目标检测模型。
- 检测类别：apple、banana、litchi、pear。
- 训练集 / 验证集 / 测试集图像数：657 / 63 / 31。
- 原始 Roboflow 标注同时包含检测框与多边形标注。训练时使用 `detect_yolo11` 中的检测专用标签，多边形标注已转换为目标检测框。

## 2. 建议汇报的准确性指标

| 指标 | 汇报时的解释 |
| --- | --- |
| Precision | 模型检出的目标中，有多少是真正正确的目标。Precision 越高，误检越少。 |
| Recall | 标注中真实存在的目标，有多少被模型检出。Recall 越高，漏检越少。 |
| mAP@0.5 | IoU 阈值为 0.5 时的平均检测精度，适合作为直观的目标检测准确性指标。 |
| mAP@0.5:0.95 | 在多个更严格 IoU 阈值下的平均精度，更能反映定位框是否准确。 |

## 5. 可直接放进汇报的文字

本项目训练了一个面向水果识别任务的 YOLO11n 目标检测模型，可检测 apple、banana、litchi 和 pear 四类目标。模型在独立测试集上取得了 `0.931` 的 `mAP@0.5` 和 `0.784` 的 `mAP@0.5:0.95`，说明模型在目标检出与定位精度上均具有较好的表现。其中 apple、litchi 和 pear 的检测精度较高；banana 类别指标相对较低，主要需要结合其测试集样本数量较少这一因素进行解释。

## 6. 汇报时建议保留的严谨说明

- 测试集共有 31 张图像，因此建议同时展示整体指标与分类别指标。
- 测试集类别分布不均衡：litchi 有 198 个目标实例，banana 只有 11 个目标实例。
- 若要解释分类别指标稳定性，优先展示测试集实例数；若要说明各类别测试覆盖比例，可展示测试集占总体比例图。
- `mAP@0.5` 更容易解释为目标检测正确性；`mAP@0.5:0.95` 更严格，更能体现检测框定位质量。
